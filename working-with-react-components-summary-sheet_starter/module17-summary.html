Please review the concepts covered this week.

Single page applications

In a single-page application, not every request for data has to come all the way from a server; instead, information is cached on the browser for quick access.

Some examples of single-page applications are:

  • Netflix
  • Facebook
  • Twitter
  • Gmail

In a SPA, all of the data retrieved from the server is cached onto the browser, so the need to go to a server every time you want a new piece of data is eliminated.

There are many benefits to SPAs, including the following:

  • They quickly display visual aspects, such as images
  • They can operate offline because of how they cache data
  • They have great overall performance; almost as good as a desktop application

However, there are some drawbacks, including the following:

  • SEO (search engine optimization) is poor
  • Security issues are more prevalent
  • The user must have JavaScript enabled for the app to run

Routing

On a single-page application, routing refers to the use of software that is built to change between different views of an application, without needing to refresh between links.

Same page linking

Same page linking encapsulates any type of linking on a single web page that does not need to go to a server to request new content.

Web vs. React components

Although they may sound similar, Web components and React components are not the same. While Web components encapsulate reusable components, React provides a declarative library that keeps the DOM in sync with your data. Web components are built with the browser’s JavaScript API, while React components are generated using the React library.
